Why this matters

AI lowers the skill floor. We focus on building local knowledge: sound logging practices, simple detection rules, and how to run safe, simulated demos so you can tell real risk from hype.

Starter labs (safe & simulated)
  1. Recon → Summarize: Run nmap -sV in a VM and use a tiny script to parse and summarize open ports into plain English. (We provide scripts in the repo.)
  2. Honeypot deploy: Spin up Cowrie in a sandboxed VM, forward non-critical ports, and collect logs to a local ELK or the supplied visualizer.
  3. Phishing recognition: Hands-on exercise generating a phishing email with AI and then dissecting the artifacts that reveal automation.
All demos are simulated and intentionally non-destructive. We never run exploits against third-party systems without explicit permission.
Quick detection checklist
  • Log unusual POST payload sizes & unknown User-Agents
  • Create simple YARA rules for commonly generated payload signatures
  • Rate-limit and fingerprint suspicious automation (we provide a logging snippet)
  • Instrument alerting for rapid exploit chaining patterns (multiple auth failures + payload variance)
